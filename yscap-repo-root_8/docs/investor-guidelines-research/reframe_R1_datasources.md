# Reframe R1 — Where a loan file's ALREADY-EXTRACTED facts live (integration points for the investor-guideline BACKEND OVERLAY)

Scope: for each condition domain, WHERE the verified/extracted data for one `application_id` lives and HOW to read it (no re-extraction). All paths are under `/home/user/yscap/yscap-repo-root_8/`.

**Big picture — there are FOUR canonical read surfaces, and an overlay should compose them (they already exist and the current `investor-guidelines/desk.js` uses exactly these):**
1. **Digital Twin** — `src/lib/underwriting/twin.js` `factsForFile(appId, client)` → canonical `loan_facts` rows. The single cross-doc fact store (borrower/entity/property/appraisal/insurance/OFAC/bank facts).
2. **Rule context** — `src/lib/conditions/engine.js` `loadRuleContext(appId)` → `{ ctx, app }`. The loan FIELDS + registered product + verified experience, normalized. This is the SAME brain the guideline layer and the existing desk use.
3. **Current document extractions** — `document_extractions WHERE application_id=$1 AND is_current AND status='analyzed'` (fields jsonb). The per-doc parsed facts; consumed by the pure desk modules (`file-view.loadContext` + `bank-liquidity`, `entity-chain`, etc.).
4. **Domain system-of-record tables + their finding tables** — `credit_reports`, `appraisals`(+`appraisal_findings`/`_comparables`/`_units`), `document_findings`, `ai_suggestions`.

The overlay's alerts should be written to **`ai_suggestions`** (domain 11) — the existing single advisory surface — NOT a new list.

## GAPS (data the overlay's guideline checks want but the file does NOT currently extract/store)

- **Seller concession %** (desk cond 3035 / `checkSellerConcession`): NOT extracted. Settlement statement is parsed (`settlement` doc_type carries `cashToClose`, `earnestMoney`, `contractSalesPrice`) but **no seller-concession field** exists; no twin key. The desk currently only finds it if a twin fact key matches `/seller.?concession/` — none is produced → always `to_verify`.
- **SOW contingency % as a CAP** (cond 2193 / `checkContingency`, 10% max): the amount IS in `tool_payload.contingency`/`subtotal` (readable via `rehab-budget.sowContingency`), but it is **not surfaced as a `sow_contingency_pct` twin fact**, so the desk's numeric check never fires. Easy bridge (compute pct from the SOW payload).
- **Liability coverage $** (cond 2186 / `checkLiabilityTier`): general-liability limit is **NOT extracted** — insurance extraction has only `dwellingCoverage` + `buildersRisk` bool. Real gap for any GL-tier guideline.
- **Zillow / market median** (cond 2798 / `checkMedianValue`): **NOT produced** anywhere (no AVM/Zillow median stored as a fact; AVM consensus in `avm-consensus.js` produces an ARV median but not a "Zillow median" fact key). The value side (ARV/as-is) IS available.
- **Flood-policy coverage amount / flood tier**: only the flood ZONE (SFHA) is known (appraisal + FEMA + twin `property.flood_zone`); no flood-insurance policy coverage $ is extracted.
- **Borrower email in rule ctx**: read from `borrowers.email` directly; not in `loadRuleContext.ctx` and not wired into the twin.

For all five signal gaps: the overlay check already **fails safe** — `desk.assessCondition` returns `to_verify` (advisory), never a fabricated conflict, when the signal is absent (`triggerApplies` is fail-open; numeric checks return `to_verify` on a null value).

## 6-line summary

1. Four canonical read surfaces already exist and the current `investor-guidelines/desk.js` composes them: the **Twin** (`twin.factsForFile`), **rule ctx** (`conditions/engine.loadRuleContext`), **current `document_extractions`** (per-doc fields), and **domain SoR tables** (`credit_reports`, `appraisals`+`appraisal_findings`, `document_findings`, `ai_suggestions`).
2. Credit (PDF+XML+parsed, `credit/index.fileCredit`), appraisal (`appraisals`/`appraisal_findings`, `appraisal/findings.computeFindings`), bank/liquidity (`bank-liquidity.assessBankLiquidity`), entity/LLC (`entity-chain.buildChain` + `llcs`), and rehab/SOW (`rehab-budget.*` + `checklist_items.tool_payload`) are all fully readable per `application_id` with concrete shapes above.
3. Fraud/OFAC individual+entity is read from `document_findings` (OFAC/background codes) plus `ai_suggestions` (assignment/collusion/identity) surfaced by `fraud-alert.fileBanner` — no re-extraction needed.
4. Loan fields come from `loadRuleContext` `ctx`/`app` (incl. normalized `note_buyer`, `is_assignment`, `rehab_type`, `program_strategy`); borrower email must be read from `borrowers.email` directly.
5. Checklist status is `checklist_items JOIN checklist_templates ON code` with satisfied = `status='satisfied' OR signed_off_at NOT NULL`; the overlay's alerts should be written to the existing **`ai_suggestions`** surface (new `source`, dedupe per note-buyer condition), never a new list.
6. True gaps (not currently extracted → checks fall back to advisory "to_verify"): **seller-concession %**, **SOW-contingency % as a fact**, **general-liability coverage $**, **Zillow/market median**, and **flood-policy coverage $** — everything else the guideline overlay needs is already stored.
